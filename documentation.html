Documentación del Proyecto: Visor de Mapas Leaflet

Resumen

Este proyecto es el Visor CAP (Consejo Agrario Provincial), un visor de mapas basado en Leaflet.js que carga capas WFS desde api.ellipsis-drive.com y ofrece varias capas base tipo IGN y satelital.

El visor incluye un header institucional con branding del CAP y un panel lateral de capas con estilos acordes a la identidad visual de la provincia.

La estructura del proyecto se mantiene en cuatro archivos: index.html, js/map.js, js/layers.js y css/styles.css.

Estructura de archivos

  • index.html: archivo principal con header institucional, contenedor del mapa y panel de capas.
  • js/map.js: inicializa el mapa, define capas base Argenmap/satelital y controla la UI de capas.
  • js/layers.js: define las capas WFS, maneja GetCapabilities, carga GeoJSON y aplica estilos y leyendas.
  • css/styles.css: estilos del visor con paleta de colores CAP (verde), header, panel de capas y leyenda.

Branding CAP

El visor incluye:

  • Header institucional con logo del Consejo Agrario Provincial (70px de altura fijo).
  • Paleta de colores: verde CAP (`#6ba043`, `#5a8f38`) para header, panel y elementos interactivos.
  • Título: "Visor CAP" en el header institucional.
  • Logo: `https://cap.santacruz.gob.ar/img/logo-dark.png`

Capas WFS

El visor actualmente expone tres capas WFS configuradas en js/layers.js:

  • Delegaciones CAP (servicio WFS principal).
  • Comisiones de Fomento (misma API WFS, con token diferente).
  • Bosques Nativos (estilo por clasificación tipo_bn y leyenda de especies).

La lógica de WFS usa GetCapabilities para obtener el primer FeatureType y luego solicita GetFeature en formato GeoJSON.

Se implementó una caché de capacidades y un prefetch inicial para acelerar la carga de capas WFS posteriores.

Mapas base

Las capas base disponibles son:

  • Argenmap (base IGN Argentina).
  • Argenmap (gris)
  • Argenmap (topográfico)
  • Argenmap (oscuro)
  • Satelital (ESRI World Imagery).

Estas capas se agregan en js/map.js usando L.tileLayer y se presentan en el control de capas de Leaflet.

Decisiones de diseño

Nota: aquí se documentan los objetivos y cambios importantes del proyecto.
  • Integrar branding institucional del CAP con header fijo y colores corporativos.
  • Mantener separación clara entre mapa y definición de capas WFS.
  • Usar capas base de Argenmap para replicar la experiencia de SITU Santa Cruz.
  • Agregar soporte visual para leyendas de clasificación en Bosques Nativos.
  • Optimizar la carga WFS con caché de GetCapabilities y prefetch.

Uso

  1. Abre index.html en un navegador moderno.
  2. El visor carga con el header del CAP y el control de capas en la esquina superior izquierda.
  3. Selecciona la capa base desde el control de capas en la esquina superior derecha.
  4. Activa la capa WFS deseada desde el panel de capas laterales (botón con icono de capas).
  5. La primera carga WFS puede tardar algunos segundos; las siguientes deben ser más rápidas gracias a la caché.

Prompts clave para replicar

Este documento resume los pasos principales realizados para llegar al estado actual del visor.

  1. Crear un visor Leaflet con estructura de archivo separada.
  2. Agregar header institucional con logo y titulo "Visor CAP".
  3. Implementar panel de capas lateral con controles WFS.
  4. Definir capas WFS en js/layers.js con URL, token y estilos.
  5. Implementar carga de WFS basada en GetCapabilities y GetFeature.
  6. Aplicar paleta de colores del CAP (verde corporativo).
  7. Mejorar velocidad con caché de capacidades y prefetch.
